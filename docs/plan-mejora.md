# Plan de mejora

## Semana 2: API inteligente y contratos de entrada/salida
- Definir un servicio o endpoint inteligente que maneje preguntas financieras, recomendaciones y extracción OCR.
- Preparar contratos claros de entrada y salida para cada operación de IA.
- Documentar cómo la app consume esos servicios.

## Semana 3: Pruebas, automatización y CI/CD
- Implementar pruebas unitarias para servicios de negocio y utilidades.
- Agregar pruebas básicas de integración para autenticación y almacenamiento.
- Configurar flujo de validación automática con herramientas de calidad.

## Semana 4: Contenedor o despliegue
- Preparar una estrategia de despliegue para la app móvil y los servicios asociados.
- Evaluar Expo EAS Build y despliegue en entornos de prueba.
- Definir una configuración reproducible para ejecución en otros equipos.

## Semana 5: Observabilidad, rendimiento y escalabilidad
- Incorporar logs de errores y eventos relevantes.
- Medir tiempos de respuesta de las consultas con IA.
- Mejorar la robustez del flujo de OCR y recomendaciones.

## Semana 6: Seguridad, documentación final y defensa técnica
- Revisar manejo de credenciales, reglas de acceso y privacidad de datos.
- Completar documentación técnica y de usuario.
- Preparar la demostración del proyecto para la defensa final.
